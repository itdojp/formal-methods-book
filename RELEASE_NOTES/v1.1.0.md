# formal-methods-book v1.1.0 リリースノート

## 概要

v1.1.0 は、Roadmap #328 の Phase 0 と Phase 1 を対象にした更新です。収録範囲は Issue #295-#301 と #313-#322 であり、Phase 2 の #323-#327 は本版に含めません。

## 章・領域別の主な変更

### 第4章 Alloy

- Alloy 6 の時相・振る舞いモデリングを追記しました。
- bounded trace semantics と、状態遷移の読み方を補強しました。
- TLA+ との比較導線、付録参照、用語整理を追加しました。
- 対応 Issue: #295

### 第7章・第8章・第12章 TLA+ / 模型検査 / 自動化

- TLA+ ツールチェーンを TLC / Apalache / Quint / CI 中心の説明へ更新しました。
- 第7章の TLA+ 時相演算子、stuttering、fairness、refinement、mapping / hiding の説明を修正しました。
- 第8章では、模型検査の保証境界、CAP / FLP の成立条件、soundness / completeness の適用範囲を精密化しました。
- 第12章では、検証成果物の CI 管理、artifact 契約、lane 分離、manifest ベースの実行保証を追加しました。
- 対応 Issue: #296, #299, #313, #314, #315, #322

### 第9章・付録F 定理証明 / AI 支援

- Lean / Rocq / LLM 支援証明の最新動向を追加しました。
- AI 支援を使う際の証跡分離、レビュー、再検証の観点を補強しました。
- 英語版の部分同期状態を translation manifest と公開 notice で追跡可能にしました。
- 対応 Issue: #298, #317

### 第10章 Rust 検証

- Rust の形式検証ツール群を追加しました。
- Aeneas を含む周辺ツールの位置付けと、CI 証跡の扱いを追記しました。
- 対応 Issue: #297

### 第13章 事例研究

- Cedar、生成AIガードレール、スマートコントラクト検証の事例を追加しました。
- 産業適用の観点から、形式手法の導入判断に使える具体例を拡張しました。
- 対応 Issue: #300

### 付録B・C・E・F、用語集、公開導線

- 付録B・C・E・Fと用語集を 2026 年版へ更新し、章末からの導線を整理しました。
- 日本語公開章を `src/ja/**` 正本から生成するようにし、公開物の drift 検出を追加しました。
- `book-config.json` / edition config からナビゲーションと公開 metadata を生成する構成へ移行しました。
- 日本語版・英語版の全 reader page を対象にした検索インデックスを実装しました。
- CHANGELOG、release notes、release 手順書を整備しました。
- 対応 Issue: #301, #316, #318, #319, #321

### クリーンアップ

- Git 管理されていた `node_modules/**`、`docs/_site/**`、旧 template 残骸、未使用 asset 群を削除しました。
- 依存取得契約を `package-lock.json` と `npm ci` に統一しました。
- 対応 Issue: #320

上記以外の tool / service は `documentation-only` です。本文では紹介していても、このリポジトリは version、実行環境、実行結果を保証しません。

## 互換性

- 公開対象は GitHub Pages 上の `docs/` と `docs/en/` です。
- 日本語 reader page は `src/ja/**` を正本とし、英語 reader page は `translation-status.json` の監査結果と公開 notice に従います。
- 実行保証 lane は Ubuntu x86_64 と GitHub Actions を基準に設計されています。`optional/manual` の Kani は PR や定期実行には含めません。
- 検索インデックスは日本語版・英語版を別々に lazy-load します。reader-facing URL の正規パスは維持します。

## 既知の制約

- 翻訳状態は 2026-07-16 時点で `partial: 22`, `synced: 1`, `stale: 0` です。完全同期された英語ページは Appendix A のみです。
- `documentation-only` の tool は紹介用であり、実行保証や固定 version の契約対象ではありません。
- Kani 0.67.0 は `optional/manual` 専用です。通常の PR / schedule lane では自動実行しません。
- PDF / EPUB の release artifact は提供しません。このリポジトリには対応する生成 pipeline が存在しません。
- Phase 2 の #323-#327 は未収録です。v1.1.0 の範囲外として継続管理します。
- 公開 provenance は source commit、Pages run、生成時刻を対応付けますが、runner imageやGitHub Actions参照の内容まで暗号学的にattestするものではありません。

## リリース時に確認すべき公開証跡

- `build-provenance.json` が `version`, `release_tag`, `source_commit`, `pages_run_id` を返すこと。
- 公開 HTML が version と source commit の meta / data 属性を持つこと。
- 英語ページの translation notice が source commit、review 日、tracking issue を表示すること。
- `translation-status-report` artifact で partial / stale の件数が release note と一致すること。

## 配布物

- GitHub Release の本文は本ファイルを正本とします。
- exact source commit と Pages run を結ぶ機械可読 provenance JSON を監査用 asset として添付します。
- PDF / EPUB は配布しません。Pages 公開物が唯一の reader-facing release artifact です。
